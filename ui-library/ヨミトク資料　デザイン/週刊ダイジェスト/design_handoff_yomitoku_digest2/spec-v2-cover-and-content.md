# 指示書v2: 週刊ダイジェスト 詳細レイアウト仕様(表紙 + 中身)

## 前提
表紙・中身の両方とも、実装(digestv4.pdf / 添付スクリーンショット)は原本と比べて文字サイズ・余白のバランスが崩れています。表紙は「カード外周の余白過多」と「要素間の縦の空白の作り方」が原因、中身は「文字サイズ・行間・パディングが全体的に一段階小さい」ことが原因です。それぞれ具体的に指摘します。

# Screen 1: 表紙ページ の修正指示

## 症状(実装 vs 原本)
1. **カード外周のグレー余白が広すぎる**。原本はキャンバスに対してカードがほぼ全面(内側マージンは上下左右とも狭い/カードが画面を大きく占有)。実装はカードの周囲に太いグレーの帯があり、カード自体が縮小されて中央に浮いているように見える。カードは「サムネイルの中の小さい画像」ではなく「画面いっぱいのポスター」として組むこと。
   - 目安: キャンバス1680×945pxに対し、カードの外側マージンは上下左右とも**24〜40px以内**に収める。実装のような100px超のグレー枠は作らない。
2. **ロゴブロックが3行になっている**(ロゴ名+タグライン+カテゴリ名)。原本は2行構成(ロゴ名+カテゴリ名のみ)。タグライン行を追加する場合は、ロゴブロック全体の高さが原本比+50%を超えないよう、タグラインは10px程度に抑え、ロゴ名との行間(margin-top)も4px以内にする。
3. **「今回は◯件の通知をまとめました」バナーと、最下部の注記(※厚生労働省…)の間に、原本にはない大きな空白ができている**。実装はこの2つの要素の間が縦にスカスカで、カード下半分が間延びして見える。
   - 修正: バナーと下部注記の間のmargin-topは**48px以内**に収める。両者の間に他の要素がないのであれば、この間隔を大きく広げすぎない(flexの`justify-content:space-between`で強制的に上下に引き離さない。原本はコンテンツを上詰めにし、下部注記もその近くに収まっている)。
4. 上記1・3の結果、全体の情報が実際より小さく・弱く・スカスカに見える。カードの外周マージンを詰め、バナー→注記の間隔を詰めるだけで密度感が原本に近づく。

## 数値仕様(表紙)
- 全体キャンバス: 1680 × 945px、背景 #fbfbfa
- 白カード: 幅1600px、高さ865px、外側マージン上下左右とも**32px**(=カードがキャンバスの94%以上を占める)
- カード内padding: 上下48px / 左右64px
- カード内の縦積み要素の間隔:
  - ロゴブロック → メイン見出し: margin-top 32px
  - メイン見出し → 対象期間ピル: margin-top 24px
  - 対象期間ピル → 件数バナー: margin-top 20px
  - 件数バナー → 下部注記: margin-top **40px(最大48px)** — ここを広げすぎない
- ロゴブロック: ロゴ名22〜32px/font-weight:800、カテゴリ名(下段)15px/font-weight:600。タグラインを入れる場合は10px/font-weight:500、ロゴ名との間はmargin-top:2px
- メイン見出し: 2行とも font-size:64px、font-weight:800、line-height:1.25(1行目 #1a1a1a、2行目 #178C7E)
- 対象期間ピル: border 2px solid #178C7E、border-radius:999px、padding:14px 28px、font-size:24px/700、color:#178C7E
- 件数バナー: background #e8f4f2、border-radius:20px、padding:24px 32px、本文26px/700(強調数字30px/#178C7E)
- 下部注記: font-size:15px、color:#666666、line-height:1.7
- 右側イラスト: 幅560px相当、右端寄せ、縦中央配置

# Screen 2: 中身(通知一覧)ページ の修正指示

**重要**: 以下の数値は目安ではなく厳守値です。Tailwindのデフォルトクラス(text-sm / text-base / text-lg など)を使わず、必ずこのpx値を明示的に指定してください。実装後は必ずリファレンス画像とスクリーンショットを並べて、文字の大きさ・太さ・行間・余白が一致しているか目視確認してください。

## キャンバス全体
- 全体キャンバス: 1680 × 945px、背景色 #fbfbfa
- 白いカード(シート): 幅1600px、高さ865px、キャンバス内で中央配置(上下左右の余白 各40px)
- カードのborder-radius: 28px
- カードのbox-shadow: 0 20px 60px rgba(20,40,38,0.08)
- カード内側の余白(padding): 上下40px / 左右48px — これより狭めない
- カード内部は縦積みflex、要素間のgap: 24px(ヘッダー→カード1→カード2→カード3→フッター注記)

## ヘッダー行
- 左グループ: メガホンアイコン付きの円 + タイトル。display:flex; align-items:center; gap:20px
  - アイコン円: 64×64px、background #178C7E、border-radius:50%、中のアイコンは白・30×30px相当
  - タイトル文字(例「今週の通知一覧（1/4）」): **font-size:38px、font-weight:800、color:#1a1a1a、line-height:1.2、letter-spacing:0**
    - NG例: 20px前後で細めのweightになっている状態 → アイコンより文字が貧弱に見える原因
- 右グループ(ロゴ): Y字アイコン40×40px + 「ヨミトク」22px/font-weight:800 + サブ文字「介護保険最新情報」12px/font-weight:600、letter-spacing:0.1em、gap 12px

## 通知アイテムカード(×3、縦に24pxずつ間隔をあけて並べる)
- 外枠: border:1.5px solid #e3e8e7、border-radius:20px
- 内側余白(padding): **上下26px、左右32px** — これより狭めない
- 横並びflex、左カラムと右カラムの間のgap: 28px

### 左カラム(幅150px固定、縦積み・中央揃え、gap:12px)
- アイコンボックス: **96×96px**、border-radius:20px、background:#e8f4f2
  - 中の線画アイコン: **44×44px**、色#178C7E、中央配置。ボックスに対して余白を持たせすぎない(アイコンがボックスの中で小さく浮かないよう、ボックス辺に対しアイコンは概ね46%の比率を保つ)
- タグpill(アイコンの下): background:#e8f4f2、color:#178C7E、**font-size:15px、font-weight:700**、border-radius:999px、padding:8px 18px、white-space:nowrap、text-align:center
  - NG例: 11px前後まで縮み、padding も4px程度に詰まっている状態

### 右カラム(残り幅いっぱい、縦積み、justify-content:space-between)
1. タイトル行: display:flex; align-items:center; gap:16px; flex-wrap:wrap
   - タイトル文字: **font-size:26px、font-weight:800、color:#1a1a1a、line-height:1.3**
     - NG例: 18〜20px程度、weightも600前後で本文とほぼ同じ大きさに見えている状態。ここが原本と最も違う点。必ずタイトルは本文の1.4倍以上のサイズ感を保つこと。
   - バッジ: **font-size:16px、font-weight:700**、border-radius:8px、padding:4px 14px
     - 重要: background #fde7e7 / color #d64545
     - 注目: background #fdf0dc / color #c9822a
     - 通知: background #e8f4f2 / color #178C7E
     - バッジはカードの右上ギリギリに張り付かせず、タイトル行の一部として自然に隣接させる(カード全体のpadding=32pxが確保されていれば自動的に離れます)
2. タイトル行と本文の間のmargin-top: 10px
3. 本文(説明、2行想定): **font-size:18px、font-weight:400、color:#4a4a4a、line-height:1.7**
   - NG例: line-height 1.3前後でほぼベタ組みになっている状態。1行目と2行目の間に明確な隙間が見える程度の行間(1.7)を必ず確保すること。
4. 本文とフッター行の間のmargin-top: 14px
5. フッター行: display:flex; justify-content:space-between; align-items:center
   - 左グループ(vol番号 + 日付、間隔28pxのgapで並べる):
     - 各アイコン: 18×18px、テキストとのgap:8px
     - テキスト: **font-size:17px、font-weight:600、color:#178C7E**
   - 右: 「資料PDFを見る →」 **font-size:18px、font-weight:700、color:#178C7E、text-decoration:underline**
     - NG例: 13〜14px程度に縮んでいる状態

## フッター注記(カード最下部)
- background:#f4f7f6、border-radius:16px、padding:16px 24px、display:flex、align-items:center、gap:14px
- 「i」アイコン: 22×22px、border:2px solid #178C7E、color:#178C7E、円形
- テキスト: font-size:15px、color:#666666、line-height:1.6

## フォント
- 日本語ゴシック(Hiragino Kaku Gothic ProN / Hiragino Sans / Yu Gothic相当)。プロジェクト標準の日本語フォントスタックを使用。
- letter-spacingは特記箇所(サブ文字 0.1em)以外はすべて 0(normal)。文字を詰めたり広げたりしない。
- font-weightは 800(タイトル類) / 700(バッジ・タグ・リンク・vol日付) / 600(サブ文字) / 400(本文説明)の4段階のみを使用し、中間のweightを使わない。

## 参考: 今回の実装(digestv4.pdf)で確認された不具合
上記のレイアウト崩れとは別に、以下も併せて修正が必要です(指示書の対象外・データ/ロジック側の問題):
- カテゴリごとにアイコンを出し分けるロジックが機能しておらず、ほぼ全項目が同じ棒グラフアイコンになっている
- 日付欄が全件「—」で空欄になっている
- 要約対象の元データに、厚労省サイトのナビゲーション構造ページ(実質的な通知内容を含まないページ)が多数混入している
