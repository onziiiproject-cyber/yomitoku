## 4. 余白・密度に関するルール（重要）

生成結果で「イラストが欠落してカード下部に不自然な空白ができる」「フッター帯の情報が詰まりすぎる」という不具合が繰り返し発生したため、以下を厳守すること。

- **イラストは必須要素として扱う。省略しない。** 本文の右側や下部にイラストを配置する設計のカード（表紙、概要ページ等）で、対応する画像が用意できない場合でも、勝手にイラストごと省略して余白を残してはいけない。必ず `/Users/onziiimac/yomitoku/design` 内から関連画像を選定して配置するか、それでも見つからない場合はイラストと同じ面積を占める控えめなプレースホルダー（グレー角丸矩形＋「イラスト：〇〇」ラベル）を必ず置き、空白のまま放置しない。
- **カード・ボックスの高さはコンテンツ量に合わせる。** テキスト量に対してカードの高さだけを大きくして中身をスカスカにするのではなく、上下パディングを32〜40px程度に統一し、コンテンツ量に応じて高さが決まるようにする。余白を埋めるために文章を水増ししない。
- **フッター帯（AIコメント／経営者への影響／確認したい方／出典など複数要素が横並びになる帯）は、最低56〜64pxの上下パディングを確保し、要素間のgapは24px以上空ける。** 詰め込みすぎて文字が小さくなる・要素が重なる状態は禁止。入りきらない場合は帯の高さを広げる（他要素の高さを削って帯を圧縮するのは禁止）。
- 1枚のスライド内で「大きな余白の空きスペース」と「情報が窮屈な帯」が同時に存在する場合は、密度の再配分（余白のある側から詰まっている側へ要素やパディングを移す）を行うこと。

## 5. 生成ルール（Claude Codeへの実行指示）

1. 上記データを受け取ったら、テーマ数に応じて `表紙(A) → 全体サマリー(B) → [概要(C) → 課題・意見(D)] × テーマ数` の順にスライドを生成する。
2. 各スライドは1920×1080pxのキャンバスに収め、文字が枠からはみ出さないよう自動で改行・文字量調整する。
3. 数値・統計・参照ページ番号などの事実情報は、必ず入力データの値をそのまま転記する（推測で数値を作らない）。
4. イラスト・アイコンは `/Users/onziiimac/yomitoku/design` フォルダの中から、スライドの文脈（サービス種別、人物像など）に最も近いものを選んで配置する。該当画像がなければ、上記トンマナに沿った控えめなプレースホルダー（グレーの角丸矩形＋ラベル）を仮置きし、後で差し替え可能なようにファイル名をコメントで明記する。
5. 各スライド末尾に必ず出典・免責表記（「厚生労働省 公開資料をもとにした自動要約です。正式な内容は原資料でご確認ください。」または該当資料名リンク）を入れる。
6. 配色・フォントサイズ・余白は本書「1. ブランド・デザイントークン」の値を厳守し、スライドごとに独自の色を増やさない。
